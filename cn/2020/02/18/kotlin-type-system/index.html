一道 Kotlin 面试题

关于程序员对 Kotlin 类型系统的理解，比较高级的一个面试问题是这样：

这段代码里面到底哪一行错了？为什么？如果某个 Kotlin 版本能顺利运行这段代码，那么如何让这个错误暴露得更致命一些？

注意这里所谓的「错了」是本质上，原理上的，而不一定是 Kotlin 编译器，IDE 或者运行时报给你的。也就是说，你用的 Kotlin 实现，IDE 都可能是错的，没找对真正错误的地方，或者没告诉你真正的原因。

（注：本文为跟风 一道 Java 面试题）
